# 🚀 Deployment Guide

## Quick Deploy to Vercel

### Option 1: Vercel CLI

```bash
# Install Vercel CLI if not already installed
npm i -g vercel

# Login to Vercel
vercel login

# Deploy to production
vercel --prod
```

### Option 2: GitHub Integration

1. Push this repo to GitHub
2. Go to [vercel.com](https://vercel.com)
3. Import project → Select repo
4. Configure environment variables in Vercel dashboard
5. Deploy

## Environment Variables (Vercel Dashboard)

Go to Project Settings → Environment Variables and add:

### Required
```
NEXT_PUBLIC_SHOPIFY_STOREFRONT_ACCESS_TOKEN=shpat_...
NEXT_PUBLIC_SHOPIFY_STORE_DOMAIN=your-store.myshopify.com
SHOPIFY_STOREFRONT_API_VERSION=2024-01

NEXT_PUBLIC_SANITY_PROJECT_ID=abc123
NEXT_PUBLIC_SANITY_DATASET=production
NEXT_PUBLIC_SANITY_API_VERSION=2024-01-01
SANITY_API_TOKEN=sk...
```

### Optional (Analytics)
```
NEXT_PUBLIC_VERCEL_ANALYTICS_ID=...
NEXT_PUBLIC_VERCEL_SPEED_INSIGHTS_ID=...
```

## Custom Domain

1. Buy domain (Cloudflare, GoDaddy, etc.)
2. In Vercel dashboard: Project → Settings → Domains
3. Add domain and follow DNS instructions
4. Enable SSL (auto-generated by Vercel)

## Troubleshooting

### Build Fails
```bash
# Check for TypeScript errors
npm run typegen
npx tsc --noEmit

# Clear cache and rebuild
rm -rf .next
npm run build
```

### Shopify Connection Issues
- Verify Storefront access token has correct scopes
- Check store domain format (include `.myshopify.com`)
- Ensure products are published to Online Store channel

### Sanity Images Not Loading
- Verify project ID and dataset name
- Check image URLs in Sanity dashboard
- Ensure CORS is configured for your domain
